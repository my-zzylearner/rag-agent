### Q3: 如何评估 RAG 效果？

**标准答案：** 常用指标：Faithfulness（忠实度）、Answer Relevancy（答案相关性）、Context Recall（上下文召回率）。工具：RAGAS 框架。

**RAG 回答：** 根据参考内容，评估 RAG 效果的方法如下：

1.  **使用工具**：
    可以使用 **Ragas** 框架来评估 RAG 效果。Ragas 是一个大模型评测框架，依赖于 ChatGPT 进行评测，因此使用前需要设置 OpenAI Key。

2.  **评估流程**：
    主要包括数据准备（将待评测数据集构建为 `dataset`）、代码实现，以及结果输出与可视化。

3.  **评估指标**：
    常用指标包括 **Faithfulness（忠实度）**、**Answer Relevancy（答案相关性）**、**Context Recall（上下文召回率）**。
    
    Ragas 提供的更详细指标及其含义如下：
    *   **Faithfulness（忠实度）**：衡量生成的答案相对于给定上下文的事实一致性。如果答案中的所有主张都能从上下文中推断出来，则视为忠实。取值范围 0-1。
    *   **Answer Relevancy（答案相关性）**：评估生成的答案对于给定提示的相关性。对于不完整或包含多余信息的答案会分配较低分数。取值范围 0-1。
    *   **Context Precision（上下文精确度）**：评估在上下文中是否所有标签相关项都被排名得更高。取值范围 0-1。
    *   **Context Relevancy（上下文相关性）**：衡量检索到的上下文的相关性。检索到的上下文应包含用于回答查询的关键信息。取值范围 0-1。
    *   **Context Recall（上下文召回率）**：度量检索到的上下文与被视为标签的注释答案的一致性程度。取值范围 0-1。
    *   **Answer semantic similarity（答案语义相似度）**：评估生成的答案与标签之间的语义相似程度。取值范围 0-1。
    *   **Answer Correctness（答案正确性）**：评估生成的答案与标签相比准确性。结合了语义相似性和事实相似性。取值范围 0-1。
    *   **Aspect Critique（方面评价）**：基于预定义的方面（如无害性和正确性）评估提交内容，用户也可以定义自定义方面。输出是二进制的。

### Q4: 什么是 LLM Agent？

**标准答案：** LLM Agent 是以大语言模型为核心推理引擎，能够自主规划、调用外部工具、循环执行直到完成任务的系统。

**RAG 回答：** 根据参考内容，LLM Agent 是以大语言模型为核心推理引擎，能够自主规划、调用外部工具、循环执行直到完成任务的系统。

它与普通 LLM 问答的区别在于：
- **普通 LLM**：输入 → 输出，一次性生成
- **LLM Agent**：输入 → 思考 → 行动 → 观察 → 思考 → ... → 输出，循环迭代

**RAG 回答：** 根据参考内容，BM25 算法的缺点主要包括以下几点：
