# 协同过滤算法技术摘要

## 核心概念
协同过滤（Collaborative Filtering, CF）通过分析用户与物品的交互历史，利用群体相似性预测用户偏好。

## 算法分类
- **UserCF (基于用户)**：计算用户间相似度，推荐相似用户喜欢的物品。
- **ItemCF (基于物品)**：计算物品间相似度，推荐与用户历史喜好相似的物品。

## 优缺点对比
- **优点**：
  - 无需领域先验知识，自动化挖掘规律。
  - 具备“惊喜”效应，可发现潜在兴趣。
- **缺点**：
  - **冷启动**：新用户或新物品因缺乏数据难以推荐。
  - **数据稀疏**：用户 - 物品矩阵稀疏影响精度。
  - **扩展性**：大规模数据下计算开销较大。

## 应用场景
适用于电商、流媒体等拥有丰富用户交互数据的平台。

# 协同过滤相似度计算方法

## 1. 余弦相似度 (Cosine Similarity)
- **原理**：计算两个用户评分向量的夹角余弦值。
- **公式**：$\cos(\theta) = \frac{A \cdot B}{\|A\| \|B\|}$
- **特点**：关注向量方向，忽略绝对数值大小，适合显式评分。

## 2. 皮尔逊相关系数 (Pearson)
- **原理**：衡量两变量间的线性相关程度，需减去各自均值。
- **公式**：基于协方差与标准差计算。
- **特点**：消除用户评分偏好（宽严）影响，适合评分尺度不一场景。

## 3. Jaccard 相似系数
- **原理**：集合交集与并集的比率。
- **公式**：$J(A,B) = \frac{|A \cap B|}{|A \cup B|}$
- **特点**：仅考虑物品是否被交互，忽略强度，适合隐式反馈。

## 总结对比
- **余弦**：侧重方向一致性。
- **皮尔逊**：侧重趋势相关性。
- **Jaccard**：侧重共同行为存在性。
- **选择**：显式反馈用余弦/皮尔逊，隐式反馈用 Jaccard。

# 协同过滤算法：基于用户与基于物品

## 1. 核心原理
- **基于用户 (User-based)**：寻找与目标用户兴趣相似的其他用户，推荐他们喜欢但目标用户未接触过的物品。
- **基于物品 (Item-based)**：寻找与目标用户已交互物品相似的其他物品，进行推荐。

## 2. 相似度计算
- **常用指标**：余弦相似度、皮尔逊相关系数、杰卡德系数。
- **作用**：量化用户行为向量或物品特征向量的重合度，构建相似度矩阵。

## 3. 优缺点对比
- **基于用户**：
  - 优点：能发现用户潜在兴趣，对兴趣变化反应快。
  - 缺点：用户基数大时计算成本高，冷启动问题明显。
- **基于物品**：
  - 优点：物品相似度相对稳定，扩展性好，适合电商等大规模场景。
  - 缺点：难以捕捉用户兴趣突变，新物品推荐能力弱。

## 4. 适用建议
- 用户兴趣多变选 User-based；物品库固定选 Item-based。
