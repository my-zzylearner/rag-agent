# 搜索排序算法详解

## 信息检索基础

信息检索（Information Retrieval）的核心任务是：给定一个查询（Query），从文档集合中找出最相关的文档，并按相关性排序。

现代搜索系统通常分为两阶段：
1. **召回（Retrieval）**：从海量文档中快速筛选出候选集（几百~几千条）
2. **排序（Ranking）**：对候选集精细排序，选出最终结果

## 经典算法：BM25

BM25（Best Match 25）是最经典的关键词检索算法，是 TF-IDF 的改进版，至今仍是全文搜索的基准算法。

### 公式
```
Score(D, Q) = Σ IDF(qᵢ) × [TF(qᵢ,D) × (k1+1)] / [TF(qᵢ,D) + k1 × (1 - b + b × |D|/avgdl)]
```

其中：
- TF(qᵢ, D)：词 qᵢ 在文档 D 中的词频
- IDF(qᵢ)：逆文档频率，衡量词的区分度
- |D|：文档长度，avgdl：平均文档长度
- k1（通常 1.2~2.0）：控制词频饱和度
- b（通常 0.75）：控制文档长度归一化程度

### 特点
- 优点：计算快，无需训练，可解释性强，对精确匹配效果好
- 缺点：无法理解语义，"汽车"和"轿车"被视为完全不同的词

## 向量检索（语义检索）

将查询和文档都转换为向量，通过向量相似度（通常是 Cosine 相似度）来衡量相关性。

### 工作原理
1. 离线：用 Embedding 模型将所有文档转换为向量，存入向量数据库
2. 在线：将查询转换为向量，在向量空间中找最近邻

### 特点
- 优点：理解语义，能处理同义词、近义词，跨语言检索
- 缺点：对精确关键词匹配效果不如 BM25，计算开销大

### 常用 Embedding 模型
- **sentence-transformers**：开源，多语言支持
- **OpenAI text-embedding-3**：效果优秀，需要 API
- **BGE 系列**：中文效果好，智源开源

## 混合检索（Hybrid Search）

结合 BM25 和向量检索的优势，是当前 RAG 系统的最佳实践。

### RRF（Reciprocal Rank Fusion）
最常用的结果融合方法：

```
RRF_score(d) = Σ 1 / (k + rank_i(d))
```

其中 k=60（经验值），rank_i(d) 是文档 d 在第 i 个检索结果列表中的排名。

### 加权融合
```
final_score = α × bm25_score + (1-α) × vector_score
```

α 通常取 0.5，可以根据任务调整。

### 为什么混合检索更好
- BM25 擅长：精确关键词、专有名词、代码、数字
- 向量检索擅长：语义理解、同义词、模糊查询
- 混合后两者互补，召回率和精度都更高

## 重排序（Reranking）

召回阶段拿到候选集后，用更精确（但更慢）的模型重新排序。

### Cross-Encoder
- 将 (query, document) 拼接后输入模型，直接输出相关性分数
- 比 Bi-Encoder（向量检索）精度高，但不能预计算，速度慢
- 适合对 top-100 候选集重排

### 常用重排序模型
- `cross-encoder/ms-marco-MiniLM-L-6-v2`（英文）
- `BAAI/bge-reranker-large`（中英文）

### 两阶段检索流程
```
向量检索 → top-100 候选 → Cross-Encoder 重排 → top-5 最终结果
```

## 学习排序（Learning to Rank）

用机器学习模型来学习排序函数，常用于传统搜索引擎。

### 三种范式
- **Pointwise**：对每个文档独立打分，转化为回归/分类问题
- **Pairwise**：学习两个文档的相对顺序（RankNet、LambdaMART）
- **Listwise**：直接优化整个排序列表（ListNet）

### 特征工程
传统 LTR 需要手工设计特征：
- 查询-文档特征：BM25 分数、TF-IDF、词频匹配数
- 文档特征：PageRank、文档长度、点击率
- 查询特征：查询长度、查询类型

## 现代搜索系统架构

```
用户查询
    ↓
查询理解（意图识别、实体抽取、查询改写）
    ↓
多路召回（BM25 + 向量检索 + 协同过滤 + ...）
    ↓
粗排（轻量模型，毫秒级）
    ↓
精排（复杂模型，Cross-Encoder 或大模型）
    ↓
业务规则（过滤、去重、多样性）
    ↓
最终结果
```

## 评估指标

| 指标 | 说明 |
|------|------|
| Precision@K | 前 K 个结果中相关文档的比例 |
| Recall@K | 前 K 个结果召回了多少相关文档 |
| MRR（Mean Reciprocal Rank）| 第一个相关结果排名的倒数均值 |
| NDCG（Normalized DCG）| 考虑相关性程度和位置的综合指标，最常用 |
| MAP（Mean Average Precision）| 多个查询的平均精度均值 |

## RAG 中的检索优化

在 RAG 场景下，检索质量直接决定生成质量：

1. **查询改写**：用 LLM 将口语化问题改写为检索友好的形式
2. **HyDE**：生成假设答案后用答案做检索，效果往往优于原始查询
3. **多路召回**：同时用多个查询或多种检索方式，取并集
4. **上下文压缩**：只返回检索文档中最相关的句子，减少噪声

# 协同过滤推荐算法

## 核心定义
基于用户历史行为数据，利用群体协作信息预测用户兴趣的推荐技术。无需分析物品内容特征，仅依赖“人”或“物”的关联关系进行推理。

## 主要类型
- **基于用户 (UserCF)**：发现兴趣相似的用户，推荐其喜欢的物品。适合社交网络及兴趣导向场景。
- **基于物品 (ItemCF)**：发现与用户已选物品相似的其他物品。稳定性更高，广泛应用于电商。

## 实现原理
1. **数据构建**：建立用户 - 物品评分矩阵。
2. **相似度计算**：使用余弦相似度或皮尔逊系数衡量用户/物品间距离。
3. **预测与排序**：加权计算预测评分，输出 Top-N 推荐列表。

## 关键挑战
- **冷启动**：新用户或新物品缺乏数据，无法计算相似度。
- **数据稀疏**：评分矩阵稀疏导致推荐精度下降。
- **扩展性**：大规模数据下计算开销大，需增量更新机制。

- **多轮对话状态跟踪**：结合上下文消除歧义。
- **槽位填充**：提取实体信息以完善意图参数。

# AI 搜索融合算法技术原理

## 1. 核心定义
AI 搜索融合旨在整合多路检索策略（如关键词、向量）与模型优化技术，以平衡召回率与准确率，解决单一搜索模式的局限性。

## 2. 关键技术原理
- **混合搜索评分 (RRF)**
  - **原理**：Reciprocal Rank Fusion。将不同排序列表映射为倒数秩并加权聚合，消除量纲差异。
  - **价值**：融合关键词（BM25）与语义向量检索，弥补单一模态盲区。
- **算子融合**
  - **原理**：将计算图中的多个底层算子合并为单一内核执行。
  - **价值**：减少显存读写开销，降低推理延迟，提升 GPU 吞吐量。
- **GEO 优化**
  - **原理**：面向生成式引擎的内容结构化适配。
  - **价值**：提高内容被大模型理解、解析与引用的概率。

## 3. 技术对比
| 维度 | 传统搜索 | AI 融合搜索 |
| :--- | :--- | :--- |
| 匹配方式 | 关键词精确匹配 | 语义 + 关键词混合 |
| 性能瓶颈 | 索引扩展性 | 推理算力与融合延迟 |

# BM25 算法原理

## 核心定义
BM25（Best Matching 25）是信息检索中的概率排序模型，用于计算查询与文档的相关性得分。它是 TF-IDF 算法的改进版本。

## 工作原理
- **词频饱和**：引入参数 $k_1$，使词频增加带来的收益呈饱和曲线，避免重复词过度主导评分。
- **长度归一化**：引入参数 $b$，根据文档长度调整得分，防止长文档因包含更多词而获得虚高分数。
- **IDF 加权**：保留逆文档频率机制，降低常见词权重，提升稀有词重要性。

## 与 TF-IDF 对比
| 特性 | TF-IDF | BM25 |
| :--- | :--- | :--- |
| 词频处理 | 线性增长 | 饱和曲线 |
| 长度偏差 | 无修正 | 有归一化 |
| 适用性 | 基础检索 | 复杂检索/RAG |

## 应用场景
常用于搜索引擎、RAG（检索增强生成）系统中的混合检索，配合向量检索提升召回精度。
